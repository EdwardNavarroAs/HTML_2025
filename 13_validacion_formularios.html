6. Buenas prácticas

  • Usa required para campos obligatorios.
  • Aplica type adecuado para cada campo (email, number, etc.).
  • Utiliza minlength, maxlength, min y max según el contexto.
  • Personaliza mensajes de error con title.
  • Evita usar solo validación HTML en contextos críticos (se recomienda validación en servidor también).
